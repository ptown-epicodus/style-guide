Style Guide

A standard for code formatting that Epicodus students may find helpful

General

These rules apply to all file types unless a more specific rule is applicable.

  • Always use single-quotes if possible.
  • Always use spaces, not tabs, for indentation.
  • Use 2 spaces for each indent

HTML

  • Use lowercase for all code.

    Example:

    <a href='http://webpage_address_using_lowercase.com' class='lowercase-class-name'>My Link</a>
    <img src='/filePath/to/Access/case-sensitive/file_system.png' alt='My Caption'>
  • Do not use in-line styling.
  • Do not include spaces on either side of '=' in tags.
  • Do not use spaces (other than in tandem with other whitespace) between content and tags.

JavaScript

  • Always use lowerCamelCase for identifiers.
  • Never exclude semicolons. (i.e. Do not use newlines to 'insert' your semicolons for you.)

    Example:

    var declaredVariable = value;
    // below is an array literal that should not be interpretted as an index of `value`
    [1].forEach(function(element) { sum += element; });
